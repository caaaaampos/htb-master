## Overview

The Droidrun CLI lets you control Android and iOS devices using natural language commands powered by LLM agents.

### Quick Start

```bash
# Setup device
htb_agent setup

# Run a command (shorthand - no "run" needed)
htb_agent "Open Spotify and play my Discover Weekly"

# Or explicit
htb_agent run "Turn on Do Not Disturb"
```

<Note>
**Auto-configuration**: If no config exists, Droidrun creates a default `config.yaml` at `~/.config/htb_agent/config.yaml` automatically.

**Command shorthand**: `htb_agent "task"` is equivalent to `htb_agent run "task"`
</Note>

## Commands

<Tabs>
  <Tab title="run">

Execute natural language commands on your device.

### Usage

```bash
htb_agent run "<command>" [OPTIONS]

# Shorthand (omit "run")
htb_agent "<command>" [OPTIONS]
```

### Common Flags

| Flag | Description | Default |
|------|-------------|---------|
| `--provider`, `-p` | LLM provider (GoogleGenAI, OpenAI, Anthropic, etc.) | From config |
| `--model`, `-m` | Model name | From config |
| `--device`, `-d` | Device serial or IP | Auto-detect |
| `--steps` | Max execution steps | `15` |
| `--reasoning` | Enable planning mode | `false` |
| `--vision` | Enable vision for all agents | From config |
| `--tcp` | Use TCP instead of content provider | `false` |
| `--debug` | Verbose logging | `false` |
| `--save-trajectory` | Save execution (`none`, `step`, `action`) | `none` |
| `--config`, `-c` | Custom config path | `~/.config/htb_agent/config.yaml` |

### Examples

<Tabs>
  <Tab title="Basic">
```bash
# Simple command
htb_agent "Open Settings"

# Multi-step task
htb_agent "Send WhatsApp to John: I'll be late"

# Specific device
htb_agent "Check battery" --device emulator-5554
```
  </Tab>

  <Tab title="LLM Providers">
```bash
# Google Gemini
export GOOGLE_API_KEY=your-key
htb_agent "Archive old emails" \
  --provider GoogleGenAI \
  --model models/gemini-2.5-flash

# OpenAI
export OPENAI_API_KEY=your-key
htb_agent "Create shopping list" \
  --provider OpenAI \
  --model gpt-4o

# Anthropic Claude
export ANTHROPIC_API_KEY=your-key
htb_agent "Reply to latest email" \
  --provider Anthropic \
  --model claude-sonnet-4-5-latest

# Local Ollama (free)
htb_agent "Turn on dark mode" \
  --provider Ollama \
  --model llama3.3:70b \
  --base_url http://localhost:11434
```
  </Tab>

  <Tab title="Advanced">
```bash
# Complex task with planning
htb_agent "Organize inbox by sender" \
  --reasoning \
  --vision \
  --steps 30

# Debug failing command
htb_agent "Book Uber to airport" \
  --debug \
  --save-trajectory action

# Wireless execution
htb_agent "Clear cache" \
  --device 192.168.1.100:5555 \
  --tcp

# Custom config
htb_agent "Enable 2FA" \
  --config /path/to/config.yaml
```
  </Tab>
</Tabs>

### `htb_agent setup`

Install Portal APK on device.

```bash
# Auto-detect device
htb_agent setup

# Specific device
htb_agent setup --device emulator-5554

# Custom APK
htb_agent setup --path /path/to/portal.apk
```

**What it does:**
1. Downloads compatible Portal APK for your SDK version
2. Installs with all permissions
3. Auto-enables accessibility service
4. Opens settings if manual enable needed

---

### `htb_agent ping`

Test Portal connection.

```bash
# Test default communication
htb_agent ping

# Test TCP mode
htb_agent ping --tcp

# Specific device
htb_agent ping --device 192.168.1.100:5555
```

**Success output:** `Portal is installed and accessible. You're good to go!`

---

### `htb_agent connect`

Connect to device via TCP/IP.

```bash
htb_agent connect 192.168.1.100:5555
```

**Prerequisites:**
```bash
# Enable wireless debugging (Android 11+)
# Settings > Developer options > Wireless debugging

# Or via USB:
adb tcpip 5555
adb shell ip route | awk '{print $9}'  # Get IP
htb_agent connect <IP>:5555
```

---

### `htb_agent disconnect`

Disconnect from device.

```bash
htb_agent disconnect 192.168.1.100:5555
```

### `htb_agent macro replay`

Replay recorded macro.

```bash
# Basic replay
htb_agent macro replay trajectories/open-settings

# Custom device and timing
htb_agent macro replay trajectories/login-flow \
  --device emulator-5554 \
  --delay 0.5

# Start from specific step
htb_agent macro replay trajectories/checkout \
  --start-from 5 \
  --max-steps 10

# Preview without executing
htb_agent macro replay trajectories/test --dry-run
```

**Flags:**

| Flag | Description | Default |
|------|-------------|---------|
| `--device`, `-d` | Device serial | Auto-detect |
| `--delay`, `-t` | Seconds between actions | `1.0` |
| `--start-from`, `-s` | Start step (1-based) | `1` |
| `--max-steps`, `-m` | Max steps to run | All |
| `--dry-run` | Preview only | `false` |

### Recording Trajectories

```bash
# Record at action level (most detailed)
htb_agent "Create alarm for 7am" --save-trajectory action

# Record at step level
htb_agent "Export contacts" --save-trajectory step
```

**Trajectory structure:**
```
trajectories/2025-10-16_14-30-45/
├── macro.json       # Action sequence
├── step_0.png       # Screenshots
├── step_1.png
└── ...
```

## Configuration

### Override Priority

1. **CLI flags** (highest)
2. **Config file** (`~/.config/htb_agent/config.yaml`)
3. **Defaults** (lowest)

### Common Patterns

<CodeGroup>

```bash Quick Test
htb_agent "Turn on dark mode" \
  --provider GoogleGenAI \
  --model models/gemini-2.5-flash
```

```bash Debug Task
htb_agent "Book ride to airport" \
  --debug \
  --reasoning \
  --vision \
  --save-trajectory action
```

```bash Cost Optimization
htb_agent "Set alarm" \
  --provider GoogleGenAI \
  --model models/gemini-2.5-flash \
  --no-vision
```

```bash Multiple Devices
for device in $(adb devices | awk 'NR>1 {print $1}'); do
  htb_agent "Clear notifications" --device $device
done
```

</CodeGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="No devices found">
```bash
# Check ADB
adb devices

# If unauthorized: Accept prompt on device
# If not listed: Try different USB port/cable
# Restart ADB
adb kill-server && adb start-server
```
  </Accordion>

  <Accordion title="Portal not accessible">
```bash
# Verify installation
adb shell pm list packages | grep htb_agent

# Reinstall
htb_agent setup

# Enable accessibility manually
adb shell settings put secure enabled_accessibility_services \
  com.droidrun.portal/com.droidrun.portal.service.DroidrunAccessibilityService
```
  </Accordion>

  <Accordion title="LLM provider errors">
```bash
# Install provider
uv pip install 'htb_agent[anthropic]'

# Check API key
echo $GOOGLE_API_KEY

# Set if missing
export GOOGLE_API_KEY=your-key
```
  </Accordion>

  <Accordion title="Command times out">
```bash
# Increase steps
htb_agent "Complex task" --steps 50

# Enable debug mode
htb_agent "Task" --debug

# Try reasoning mode
htb_agent "Multi-step task" --reasoning
```
  </Accordion>

  <Accordion title="TCP connection fails">
```bash
# Enable TCP mode (USB connected first)
adb tcpip 5555

# Get device IP
adb shell ip route | awk '{print $9}'

# Connect
htb_agent connect <IP>:5555

# Verify
htb_agent ping --tcp
```
  </Accordion>
</AccordionGroup>

$env:GOOGLE_API_KEY="your-key"
